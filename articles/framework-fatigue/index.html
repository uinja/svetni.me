JavaScript Framework Fatigue: A Never-Ending Story

Written by Miss Baker

Remember when there were like 3 frameworks?
Now there are 300 and each is the "best" according to its creator.

A Brief History of JavaScript Frameworks, With Commentary

2006 — jQuery
Write less, do more. We wrote so much more. It was a golden age. We were naive.

2010 — Backbone.js
Structure! Models! Collections! We were learning things. We did not know how much there was to learn.

2012 — Angular 1
Google made it. It must be good. We adopted it completely. Two-way data binding felt like magic.
(Note: The magic was a memory leak.)

2013 — React
Virtual DOM! Unidirectional data flow! JSX, which we hated for one week then accepted completely!
This one actually changed things.

2016 — Vue
Gentle learning curve. Friendly docs. Approachable to beginners. Everyone liked it. This made React developers nervous.

2019 — Svelte
"What if there was no virtual DOM?"
Compiler magic. Less runtime. Compelling benchmarks. A devoted community who speaks of it with quiet reverence.

2020 — Everything at Once
Solid. Qwik. Astro. Remix. Next. Nuxt. Analog. Start. Fresh. Island architecture. Partial hydration. Progressive enhancement. Resumability.

A new framework ships approximately every 72 hours. This is not a metaphor.

Choosing a Framework: The Modern Process

  1. Research for three weeks
  2. Post to Twitter asking for opinions (mistake)
  3. Receive 200 opinions, all confident, all contradictory
  4. Choose one based on which documentation website had the nicest palette
  5. Go all in: component library, testing setup, CI pipeline
  6. The author of a different framework publishes a benchmark that makes yours look bad
  7. Spend two days reading the benchmark methodology
  8. The methodology is sound
  9. Stay anyway. Migration is harder than acceptance.

What Nobody Tells You

  • The framework is not the hard part. The ecosystem is the hard part.
  • A maintained framework with a decent community beats a theoretically perfect framework with three contributors.
  • jQuery still powers a meaningful percentage of the internet and we don't talk about it.
  • Rewrites are almost always worse than they needed to be.
  • You will be maintaining this choice for 4–7 years.
  • Choose boring technology for production, exciting technology for side projects.
