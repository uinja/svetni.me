Pair Programming: Closer Than a Marriage

Written by Miss Baker

Nothing reveals your true self like pair programming.
Your partner will learn things about you that therapists charge for.

The Setup

We were assigned a "simple" feature. Authentication refactor. Should take one day.
We blocked out two days, to be safe.
We needed four.

Hour One: The Honeymoon Phase

Everything is collaborative. We're sharing insights. Someone makes coffee.

We discover immediately that we have completely different vim/emacs/VS Code opinions.
We spend 25 minutes deciding which machine to use.
We use mine. My partner pulls up their custom .zshrc and I see 47 aliases I don't recognize.

Hour Two: Philosophical Divergence

We hit the first real function. I want it to return early on errors. They want guard clauses at the top.
Both are valid. We code it both ways to compare.
We have a 30-minute conference comparing the two approaches.
We use neither. We write a third version we both dislike slightly less.

Things I discovered about my pair partner:

  • They think a function should never exceed 20 lines. Our function is 47 lines. It took an hour.
  • They name variables using '80s song lyrics. totalEclipseOfTheCart. I've started doing it too.
  • They eat cheese while coding and it bothers me an irrational amount.
  • They actually ENJOY refactoring. They smile. It scares me.

Things they learned about me:

  • I mutter to myself in three languages when debugging.
  • I rename the same variable five times before committing, then usually revert to the first name.
  • I hold the delete key like it owes me money.
  • I have strong opinions about bracket placement that I was not previously aware of.

Hours Three and Four: The Crucible

We are no longer talking about code. We are resolving something deeper.
Themes include: autonomy, communication, what constitutes "clean."
We eat lunch without pausing. This is the closest humans get to flow state.

By hour three, the code is actually quite good. Neither of us admits this.
By hour four, we've written tests. Real tests. We're both surprised.

The Aftermath

By hour six, we decide to never pair program again.

Next week: mandatory pair programming session #2.
Same partner. Different feature. Longer.

We've pre-scheduled a 30-minute decompression walk afterward.
We're bringing snacks. And noise-cancelling headphones. Individuals.

Strangely, the code from session one is the best code either of us has shipped this quarter.
We will not be discussing this.
