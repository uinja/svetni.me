Linting and the Meaning of Life

Written by Miss Baker

Our linter now has more opinions than the entire product team combined.

The Linter, as a Personality

If the linter were a person, it would be the colleague who:

  • Corrects your grammar in Slack messages
  • Sends a detailed email about the subject line of your last email
  • Has opinions about the font in your slide deck
  • Never says anything wrong, technically
  • Makes you want to quit on good days

It is somehow both incredibly useful and deeply personal.

A Session with My Linter

I write a function. It is functional. It does what it's supposed to do. I feel good about it.

I save the file.

83 warnings.

Breakdown:

  • 41 warnings about unused import path (I was going to use it)
  • 23 warnings about comma placement (strong opinions, lightly documented)
  • 12 warnings about missing return types (fair, honestly)
  • 4 warnings about variable naming conventions
  • 2 warnings about function length
  • 1 existential: Unexpected console statement (it knows I'm debugging and it disapproves)

Disabling Rules: A One-Act Play

Me: "Can we disable the no-explicit-any rule? The third-party library doesn't have types."

Linter config (via CI): This rule cannot be disabled globally. Use inline suppression with a comment.

Me: // eslint-disable-next-line @typescript-eslint/no-explicit-any

Reviewer, in code review: "We should avoid suppressing lint rules."

Me: [does not respond]

The Semicolon Incident

I changed one semicolon to a colon (typo). This triggered:

  • 1 lint error
  • 1 pre-commit hook failure
  • 1 CI pipeline run (34 minutes)
  • 1 Slack message from CI bot
  • 2 confused DMs from teammates who saw the CI failure
  • 1 PR comment: "Did the build break?"
  • 34 minutes of my life

The build passed. The colon became a semicolon again. The universe continued.

Conclusion

I believe in the linter. Consistently formatted code is better for everyone.
The linter is correct, statistically.

I also believe that 79 comma warnings in a single file is a form of feedback that damages the soul.

These beliefs coexist. Software engineering is a discipline of contradictions.
