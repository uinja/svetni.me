Documentation: A Work of Fiction

Written by A dev

Our codebase has documentation. Let me be clear about what that means:
It exists. It was written at some point. It lives in our wiki.

The Documentation Ecosystem

We have documentation in the following locations:

  1. The wiki. Last updated by someone who no longer works here.
  2. The README. Accurate as of Node 12. We are on Node 20.
  3. Inline comments. Written by an optimist in 2018. The optimist has since left.
  4. A shared Google Doc. Open to edit. Someone edited it with just the word "why." We don't know which part they meant.
  5. Confluence. Yes, we have Confluence. Nobody finds anything in it. This is by design, apparently.

Documentation vs Reality: A Comparison

What the docs say What actually happens
"Install with npm install" Install, then fix peer dependency conflict, then downgrade Node, then pray
"Configuration is straightforward" The config file is TOML with JSON-style overrides. Nobody knows why.
"See the examples folder" Examples folder deleted during a cleanup in 2021, referenced in 14 docs
"API is self-explanatory" API changed 6 times. Docs reflect version 1. You are on version 4.
"Authentication is handled automatically" You must pass a token, also a secret, also a timestamp that expires in 30 seconds
"Default values are sensible" Default timeout is 0ms. This is undocumented. Good luck.

The Improvement Initiative

Last quarter, we launched "Docs Week." Every engineer dedicated half a day to writing documentation.

Results:

  • Three new pages explaining what main.js does. Each contradicts the others.
  • One brilliant diagram that nobody can read at zoom levels under 400%.
  • Seventeen TODO comments elevated to wiki status.
  • A new section called "FAQ" containing one question ("Why does this exist?") with no answer.

Docs Week has been rescheduled to next quarter.

What We Actually Do

At this point, our real documentation is:

  • Slack search
  • The person who built the thing, if they're still here
  • Stack Overflow, 2019–2022 vintage
  • Trial and error, with emphasis on error

I once found the correct configuration by reading a closed GitHub issue from a stranger in 2020 who had given up and deleted their comment, but the edit history survived.

That was a good day.
