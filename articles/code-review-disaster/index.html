Code Review: A Masterclass in Passive Aggression

Written by Miss Baker

Nothing says "I respect your work" like a code review with 47 comments.
Each one carefully crafted to question your life choices without explicitly doing so.

The Comment Taxonomy

After years of research, I have classified code review comments into five species:

1. The Polite Knife

"Interesting approach to spacing."

This person has seen the inside of your function. They are horrified. They are too professional to say so directly. The word interesting is doing extraordinary work in this sentence.

2. The Escalating Question

"Why did you choose this name?"

This is not a question. This is an indictment. The correct response is to immediately rename the variable and never speak of it again.

3. The Deferred Horror

"Let's discuss this in person."

The code is so bad it cannot be addressed in text. The reviewer needs eye contact, possibly a whiteboard, and ninety minutes that neither of you has.

4. The Passive Suggestion

"Could we make this more idiomatic?"

No. We could not. We are on a deadline. This works. The idiom can wait. It will not wait.

5. The Existential Review

"I left some thoughts."

The thoughts number forty-seven. Three concern logic. Forty-four concern variable naming, line length, and a comment I left in 2021 that apparently still troubles them.

What I Was Thinking (A Confession)

Comment: "This LGTM but the nesting here is a bit deep."
My inner monologue: It's 11 PM. I nested because I was tired. The tests pass. What do you want from me.

Comment: "Have you considered extracting this into a helper?"
My inner monologue: Yes. I considered it. I decided against it. I regret nothing. I will extract it now and resent you gently.

Comment: "Not blocking, but..."
My inner monologue: Everything after 'not blocking, but' is blocking.

The Reapproval Dance

After forty-seven comments, I address forty-six.
The forty-seventh I cannot reproduce.
I comment: "Cannot reproduce, please advise."
They reply: "Ah, no longer relevant. Looks good!"

Thirty-six review cycles total.
PR approved.
Deploys cleanly.

That specific commit is now the foundation of a key billing flow.
I will not be taking questions.
