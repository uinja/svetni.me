Merge Conflict Epic: Return of HEAD

Written by Miss Baker

I opened a PR and immediately met an old enemy: conflict markers.

The Setup

I'd been on a feature branch for nine days.
The team had been on main for nine days.
We were not communicating about this.

git merge main

43 conflicts.

Act I: The Confrontation

<<<<<<< HEAD
const total = calculateTotal(cart, discount);
=======
const total = computeCartTotal(cart, user.discount, tax);
>>>>>>> main

<<<<<<< HEAD stared into my soul.
======= separated hope from consequences.
>>>>>>> main mocked my optimism.

Someone had renamed calculateTotal during my nine-day absence.
Someone had also added tax support, which I did not know about, which fundamentally changes the output of any calculation I wrote.

This was going to be a long afternoon.

Act II: The Resolution (Attempted)

Conflicts 1–15: Mechanical. Different naming, same intent. Resolvable in minutes. I am optimistic.

Conflicts 16–28: Structural. They extracted a utility I had also extracted, but into a different file with a different name. I pick theirs. I update my imports. Three new conflicts appear.

Conflicts 29–37: Philosophical. The tests were reorganized. My tests test things that were moved. The test structure assumes a shape that no longer exists. I start rewriting tests.

Conflicts 38–43: I stop understanding what I'm looking at. I take a walk.

Act III: The Twist

Two hours later, the merge is complete. Tests pass. I open a PR.

I review my own PR.

I notice my version of the billing calculation does not apply the tax.
Theirs did.
I chose mine in conflict 4, assuming it was equivalent.
It was not equivalent.

The conflict was gone.
The bug was invisibly introduced.
CI passed because the tests didn't test tax scenarios.

Somebody caught it in code review, seven days later, while working on something else.

Epilogue

The lesson: never resolve a conflict without understanding both sides.
The practice: always run a broader test suite after merge.
The reality: we added a tax scenario test.

I now merge main every two days while on a feature branch.
I would merge it every morning but I'm not ready to go that far.
