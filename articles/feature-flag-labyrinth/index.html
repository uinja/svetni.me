Inside the Feature Flag Labyrinth

Written by A dev

We added feature flags to reduce risk.
Now we have 47 flags and no one knows the default state.

Why We Did This

Feature flags were introduced as a best practice. A blog post recommended them. The blog post was correct.

The idea: deploy code to production but keep it hidden behind a flag. Enable for 1%, then 10%, then everyone. Safe. Gradual. No big-bang releases.

In theory this is elegant. In practice we now have a system that requires a spreadsheet to understand.

The Flag Inventory

We have 47 active feature flags. Their names describe a taxonomy of ambition:

  • new-dashboard (dashboard shipped 8 months ago, flag never cleaned up)
  • new-dashboard-v2 (same dashboard, different experiment, also shipped)
  • new-dashboard-v2-rollback (we rolled back briefly, this flag was created then, it is now always-on)
  • new-dashboard-v3 (ongoing)
  • enable-checkout (this is not new, checkout exists, the flag predates our memory)
  • beta-checkout (related to above, unknown relationship)
  • experimental-checkout-flow (also related, the relationship is a mystery)
  • notifications-v2 (v1 flag not found)
  • enable-analytics-tracking (off in prod for legal reasons, on in staging, this is backwards)
  • dark-mode (users requested this in 2021, flag is still false for 60% of users)

The Bug Report

A user reported a specific, reproducible bug. After investigation, we narrowed it down:

Bug only occurs when:

  • new-dashboard-v3 is on
  • beta-checkout is off
  • notifications-v2 is set to gradual (10%)
  • It is Tuesday
  • The user is in UTC-3

Wednesday: cannot reproduce.
Thursday: still cannot reproduce.
Monday: cannot reproduce.
Tuesday, 9 AM UTC-3: reproduced, fixed.

The Cleanup Initiative

Once a quarter, someone proposes a Feature Flag Cleanup Sprint.

The procedure:

  1. List all flags
  2. Find flag owners (many no longer work here)
  3. Determine if flag can be removed
  4. Remove it...probably

We have held four cleanup sprints. Net flags removed: 7. Net flags added during the cleanup sprint: 9.

Current Policy

All new flags must include:

  • A description
  • An owner
  • An expiry date

Expiry dates are not enforced.
Owners change when people change teams.
Descriptions are written at 4:30 PM on a Friday.

We have 47 flags.
Next quarter: Cleanup Sprint #5.
