CSS Specificity Wars

Written by A dev

At 10 AM, .card .title worked.
At 10:07, someone introduced .page .content .card .title.
By lunch, we had #app .page .content .card .title.urgent.

How We Got Here

It started, as it always does, with a very reasonable request.

"The title on the card in the dashboard looks different than the title on the card in the modal. Can we fix it?"

We fixed it. We used a more specific selector. This was the first domino.

The Escalation

Day 1: Someone needs the .card .title to be red in the sidebar only. They add .sidebar .card .title. Clean.

Day 3: The modal card title also needs to be red, but only on mobile. We get .modal .card .title and a media query.

Day 5: Someone overrides the sidebar style for a new feature. They add .sidebar.new-feature .card .title. The old style stops working in a different context. They add !important.

Day 7: Two !important declarations are now fighting each other. One wins based on source order. Nobody knows which file loads first.

Day 9: An engineer new to the codebase adds .card .title { color: inherit; } to solve a different problem. Everything is blue.

Day 11: #app .page .content .card .title.urgent is added by someone under deadline pressure. It works. It cannot be overridden by anything without an id selector and prayer.

The Peace Treaty (Attempt 1)

We held a CSS governance meeting (yes, this is a real thing we did).

Proposed rules:

  • No ID selectors for styling
  • Max specificity: two classes
  • !important requires a comment explaining why and peer review
  • New feature styles go in their own file

The treaty was added to the contributing guide.
The contributing guide is not required reading.
The contributing guide has not been read by anyone who joined in the last eighteen months.

The Peace Treaty (Attempt 2: BEM)

We renamed everything using BEM: .card__title, .card__title--urgent.

This worked for six weeks.

Then someone wrote a component that needed .card__title to behave differently inside a .card--featured context. They added .card--featured .card__title. The BEM was compromised. It was accepted as normal.

Now we have both systems coexisting. Legacy specificity wars with modern BEM in occasional skirmishes.

Current state of our CSS: functional, mostly, if you know where to look.
