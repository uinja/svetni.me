Highlights how techniques like MoE, Batching, and KV Cache compression (MLA) drive down production costs. Argues that subscription models ($20/mo) are loss-leaders for intensive "agentic" users.

## Fact Synthesis & Notes
- **Core Message:** The cost of AI isn't just about "intelligence," it's a structural battle against memory bandwidth limitations.
- **Key Facts:**
    - H100 node cost: ~$32/hour.
    - Token production cost (marginal): ~$0.10 - $0.40 per 1M tokens with optimal batching.
    - Bottleneck: Loading model weights into HBM (high-bandwidth memory).
    - MoE (Mixture-of-Experts) reduces compute/memory load by activating only relevant subsets.
- **Editorial Strategy:** Explain the "invisible" reason why AI feels expensive but is getting cheaper. Use the term "проблемната област на паметта" (memory bottleneck) to explain the technical hurdle. Contrast API vs Subscription models.
