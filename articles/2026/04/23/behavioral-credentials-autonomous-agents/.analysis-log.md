## Text from Original Article

> ...
> Authorization answers a second: What is it allowed to access? Autonomous agents introduce a third: Does it still behave like the system that earned that access? That third question is the missing layer.

## Fact Synthesis & Notes
- **Core Message:** Traditional static authorization (credentials/tokens) is insufficient for autonomous agents because their behavior can "drift" over time without any external changes or breaches.
- **Key Concepts:**
    - **Behavioral Identity:** The runtime profile of how an agent makes decisions (consistency, confidence, tool-use patterns).
    - **Black Box AI Drift:** Cumulative shifts in behavior due to interaction patterns, memory, and context.
    - **Behavioral Attestation:** A proposed architecture where trust is continuously re-earned at runtime based on telemetry.
- **Editorial Strategy:** Highlight the mismatch between stable software assumptions and dynamic agent behavior. Emphasize the need for "behavioral telemetry" as a runtime control layer.
