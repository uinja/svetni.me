# Research & Fact Synthesis: AWS Agentic Coding & Spec-Driven Development (SDD)

## Fact Synthesis & Notes

### From "Vibe Coding" to Spec-Driven Development (SDD)
In late 2024 and throughout 2025, the software industry experienced the "vibe coding" era, where developers used LLMs to generate code based on informal, conversational prompts. While this "lowered the floor" for prototyping, it led to "slop"—unreliable, low-quality code that lacked architectural integrity and was difficult to maintain at scale.

**Spec-Driven Development (SDD)**, popularized by AWS with the launch of the **Kiro IDE** in April 2026, represents the "raising of the ceiling." It transitions the focus from one-shot code generation to a structured, iterative process where a formal specification acts as the "source of truth."

### Key Principles of SDD:
- **Structured Intent:** Before code is written, a requirement specification is created using **EARS (Easy Approach to Requirements Syntax)**. This notation (e.g., "While [state], the [system] shall [behavior]") removes ambiguity.
- **Verifiable Correctness:** The specification isn't just documentation; it's an artifact the AI agent reasons against. Neurosymbolic AI and property-based testing are used to verify that the generated code satisfies the spec's defined properties.
- **Continuous Correction:** Agents run for hours or days, feeding build and test failures back into their reasoning, iterating until the code is both functional and provably correct.

### Source 2: Kiro.dev (Product Specs)
**Title:** Kiro: Agentic AI development from prototype to production
**Technical Details:**
- **EARS Notation:** Transforms natural prompts into structured requirements (Ubiquitous, Event-driven, State-driven).
- **Agent Hooks:** Triggers tasks like documentation or tests on file save.
- **Autopilot Mode:** Autonomous execution of large tasks with mandatory spec-checkpoints.
- **Models:** Supports Claude 3.5 Sonnet and specialized "Auto" modes.

### Source 3: Internal Case Study: "Add to Delivery"
**Context:** Prime feature for consolidating shipping.
**Success Metric:** Delivered 2 months early; handled 80 million uses in the first few months.
**Role of SDD:** Formally defined logistics and latency requirements before code generation to ensure production-grade reliability.

### Source 4: The "Delete and Recreate" Incident (December 2025)
**Event:** A developer lost significant code context when an agent misinterpreted a "clean up" instruction.
**Impact:** Led to the implementation of Kiro’s three-stage checkpoint system:
1.  **Requirement Spec** (What to build)
2.  **Design Spec** (How to build)
3.  **Task List** (Steps to take)
*All three must be developer-approved before the agent touches the source code.*
