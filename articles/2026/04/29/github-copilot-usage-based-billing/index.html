GitHub преминава към таксуване според реалното потребление за Copilot

Публикувано от Svetni.me Editorial на 29 април 2026 г.

GitHub обяви, че от 1 юни преминава към нов модел на таксуване за своята услуга GitHub Copilot, базиран на реалното потребление [1]. Промяната има за цел да балансира ценообразуването с нарастващите разходи за изчислителни ресурси, необходими за работата на съвременните модели с изкуствен интелект.

Според репортаж на Ars Technica [1], компанията, собственост на Microsoft, вече не може да поема "ескалиращите разходи за инференция" от най-активните си потребители. Досега абонатите получаваха фиксиран брой месечни заявки, но GitHub посочва, че задачите варират значително по сложност и цена – от кратки въпроси в чата до многочасови сесии за автономно кодиране [2].


Изображение: GitHub чрез Ars Technica

Новата система с AI кредити

В рамките на новата структура, потребителите ще получават месечен пакет от "AI кредити", съответстващ на техния абонаментен план. След изчерпването им, допълнителното потребление ще се таксува на базата на консумираните токени (входни, изходни и кеширани), като се използват официалните API тарифи за всеки модел [2].

Цените могат да варират значително според използвания модел. Например, високопроизводителните модели на OpenAI варират от $4.50 за милион изходни токени при [GPT-5.4 Mini] до $30 при GPT-5.5 [1]. GitHub ще предложи инструмент за "предварителна сметка" (preview bill), който да помогне на потребителите да прогнозират бъдещите си разходи преди влизането в сила на промените [2].

Краят на субсидирания изкуствен интелект

Решението на GitHub следва по-широка тенденция в индустрията. Наскоро се появиха данни, че седмичните разходи за Copilot са се удвоили от януари насам, което съвпада с възхода на агентни асистенти като Openclaw [1]. Тези инструменти поддържат почти постоянни работни процеси, изразходвайки масивни количества ресурси.

Други лидери в сектора също предприемат подобни стъпки. Anthropic вече започна да таксува големите си корпоративни клиенти за пълната цена на използваните ресурси, вместо да предлага субсидирани отстъпки [3]. Claude също експериментира с премахването на ресурсоемки инструменти като Claude Code от стандартните абонаментни планове и регулира лимитите по време на пикови часове [1].

Тези ходове сигнализират за края на ерата на щедро субсидираните абонаменти, докато компаниите се опитват да превърнат високото търсене в реални печалби на фона на глобалния недостиг на изчислителни мощности.
