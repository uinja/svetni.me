Xiaomi пусна MIMO Code: Инструмент за кодиране с ИИ, който превъзхожда Claude Code

Публикувано от Svetni.me Editorial на 15 юни 2026 г.

Езиковите модели за писане на код често страдат от прогресивно влошаване на качеството при дълги работни сесии. Когато контекстният прозорец се препълни, по-ранните решения и текущото състояние на проекта биват компресирани или изгубени, което принуждава разработчиците отново да обясняват контекста на своите проекти.

За да се справи с този проблем, екипът по изкуствен интелект на Xiaomi пусна с отворен код проекта MiMo Code V0.1.0 [1]. Това е терминален ИИ асистент за програмиране, който според вътрешни тестове на китайския гигант превъзхожда Claude Code при сложни, многостъпкови задачи с над 200 изпълнителни стъпки [1].


Авторско изображение: Управление на състоянието и паметта чрез допълнителен подагент в MiMo Code

Архитектура на паметта от четири слоя

Вместо да разчита на по-добро компресиране на контекста, MiMo Code внедрява изричен механизъм за съхранение и извличане на информация в реално време, захранван под капака от модула за търсене FTS5 на SQLite [1]. Паметта е разделена на четири слоя:

  1. Проектна памет: Управлява се чрез персистентен файл MEMORY.md.
  2. Сесийни контролни точки (Checkpoints): За запазване на състоянието на сесията.
  3. Работни бележки (Scratch notes): Бързи временни бележки за текущата логика.
  4. Прогрес лог за всяка задача: Хронология на изпълнението.

Ключът към тази система е разделението на труда. Докато основният кодиращ агент (fork на платформата OpenCode) се фокусира върху писането на код, независим подагент („checkpoint-writer“) работи паралелно като архитект [1]. Той следи действията на основния агент и обновява чертежите и бележките в реално време. Когато контекстът на модела наближи лимита си, системата автоматично реконструира работната среда от запазените контролни точки, спестявайки нуждата от повторно въвеждане на контекст.

Резултати и сравнение на моделите

Придружен от новия водещ модел на компанията MiMo-V2.5-Pro, MiMo Code показва силни резултати на ключови бенчмаркове в сравнение с Claude Code (комбиниран с Claude Sonnet 4.6) [1]:

  • SWE-bench Verified: 82% срещу 79% за Claude Code.
  • SWE-bench Pro: 62% срещу 55% за Claude Code.
  • Terminal Bench 2: 73% срещу 69% за Claude Code.

Интересното е, че самият софтуерен облик (harness) носи около 5% от тези подобрения. Когато един и същ модел се стартира в двете среди, версията на Xiaomi печели благодарение на оптимизираното разпределение на контекста. При потребителски тестове с 576 разработчици, двата инструмента се представят сходно при задачи под 200 стъпки, но над този праг MiMo Code постига над 65% успеваемост [1].

Достъпност и пазарна стратегия

За да привлече разработчици, Xiaomi пусна MiMo Code под лиценз MIT и предлага безплатен (за ограничен период) достъп до модела MiMo-V2.5, поддържащ контекстен прозорец от 1 милион токена [1]. Проектът се ръководи от Фули Луо, ветеран от DeepSeek [1]. Стратегията следва познатия модел на ниските цени за висококачествени услуги с отворен код с цел бързо изграждане на мащабна екосистема.

Софтуерът позволява лесно прехвърляне на MCP сървъри и конфигурации от Claude Code и поддържа работа с външни модели от други доставчици като DeepSeek, Kimi и GLM [1].
