Cache Me If You Can

Written by A dev

Nothing builds character like debugging stale cache in production.

The Incident

We deployed a critical fix at 2:14 PM.

At 2:17 PM, we received a message: "It's still broken."

This began a three-hour journey through the distributed memory of our infrastructure.

Layer 1: Browser Cache

We told users to hard refresh. Half of them don't know what that means. One user interpreted it as restarting their computer. The bug, from their perspective, was fixed. We moved on.

Layer 2: CDN Cache

The CDN confirmed: it was caching the old response. TTL: 1 hour. We deployed 43 minutes ago.

We purged the cache. CDN confirmed success. Users reported the bug persisted.

We purged the cache again. CDN still confirmed success. Bug persisted.

After the third purge, we discovered we had been purging the staging CDN. The production CDN has a different dashboard URL that differs by exactly one hyphen.

Layer 3: The Application Cache

Our application has an in-memory cache. TTL: 15 minutes. The cache key included a value that hadn't changed, so the fix was being cached over by the old, unfixed version.

We cleared the application cache on all three nodes.

We had four nodes. The fourth node was in a different availability zone and had not received the memo.

Layer 4: The Edge Case

By 4:45 PM, 97% of requests were going through correctly. A handful of users in a specific region were still experiencing the bug.

We checked the regional CDN configuration.

There was a regional CDN configuration. We were unaware it existed. It was serving cached content from two days ago, including a version of the page that predated both the bug and the fix.

We cleared it. We celebrated.

What We Learned

  1. Document every caching layer. We now have a diagram. It has eight layers.
  2. Purge all levels simultaneously. We now have a runbook. It has twelve steps.
  3. Test cache invalidation as part of every deploy. We have added this to the checklist.
  4. Verify which environment's cache you are purging before purging.

Item 4 has been added in bold.

The cache system now works correctly.
We don't fully understand why.
This is fine.
