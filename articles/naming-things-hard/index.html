Naming Things Is Still Hard

Written by A dev

We spent 45 minutes naming one helper function.

The Function in Question

The function takes an array of user objects, filters inactive ones, formats the names, and returns a sorted list. Straightforward.

Naming it should take thirty seconds.
It took forty-five minutes.
This is a true story.

The Candidates

processData
Classic. Meaningless. Accepted wisdom says this is bad. We reject it immediately.

processUserData
Better. Still vague. What kind of processing? We table it.

getUsers
Someone argues this implies a data fetch. It doesn't fetch. We reject it.

filterAndFormatUsers
Accurate. Slightly long. Someone says it violates the single-responsibility principle. The function does two things. This is a fair point. We argue about it for six minutes.

prepareUserList
Prepare for what? Nobody knows. We move on.

buildUserDisplayList
This one has momentum for a moment. Then someone points out it's called from a non-display context. Defeated by scope concerns.

getUsersForDisplay
Same objection. Not always for display. Rejected.

normalizeUsers
Normalize feels clinical. Is it accurate? Yes. Does it feel like it? No.

formatActiveUsers
Fairly accurate. "Sorted" is implicit. Someone asks if sorting counts as formatting. We spend four minutes on this.

The Decision

We chose processDataCore.

Nobody is sure how we got there. The meeting notes reflect only the outcome.

Two days later, a similar function was needed for admin users. It was named processDataCoreNew.

processDataCoreNew is currently used in eleven places.
processDataCore is currently used in nine places.
Neither name tells you what the function does.

The Philosophy

There are two hard problems in computer science:

  1. Cache invalidation
  2. Naming things
  3. Off-by-one errors

The naming problem is not fundamentally technical. It is linguistic, social, and organizational. It reveals disagreements about abstraction, responsibility, and what code is for.

We are building a glossary.
The glossary has twelve entries.
Nobody uses it.
It is, itself, named somewhat ambiguously.

We will revisit this next sprint.
